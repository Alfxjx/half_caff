1 产品概述

HalfCaff 是一款基于药代动力学（Pharmacokinetics）的个性化咖啡因管理工具。 它通过基于用户参数的科学估算，帮助用户追踪体内咖啡因残留、预测代谢趋势、优化饮用时机， 从而在享受提神效果的同时，尽量降低对睡眠和健康的潜在影响。

🧮

药代动力学引擎

一级消除动力学 · 一室模型 · 个性化半衰期

📊

实时可视化

24h 趋势曲线 · 入睡风险提示 · 保守建议

🧬

CYP1A2 基因型 · 8大影响因素 · 代谢测试

核心价值主张： 让每一杯咖啡都更可控 —— 在提神效果与晚间休息之间做出更稳妥的选择。

4 核心数据模型

UserProfile · 用户档案

interface UserProfile {
  dailyBudget: number;       // 日安全上限 (mg), 默认 400
  targetSleepTime: string;   // 目标入睡时间, 默认 "23:00"
    metabolismPreset: 'fast' | 'normal' | 'slow' | 'unknown';
    customHalfLife?: number;   // 可选人工覆盖；如有值则优先使用
  isSmoker: boolean;         // 是否吸烟
  usesOralContraceptives: boolean;  // 口服避孕药
  isPregnant: boolean;       // 是否怀孕
  liverDisease: 'none' | 'mild' | 'moderate' | 'severe';
  age: number;               // 年龄
  drinksAlcohol: boolean;    // 是否饮酒
    cyp1a2Genotype?: 'AA' | 'AC' | 'CC' | 'unknown';
}

说明：计算后的半衰期不直接持久化，统一在运行时由 calcHalfLife(profile) 推导，避免模型出现双重真相。

5 科学计算公式

公式 1 · 单次摄入后剩余量

A(N) = X × (1/2)^(N / t₁/₂)
A(N)
N时刻体内量(mg) X
摄入剂量(mg) N
经过时间(h) t₁/₂
消除半衰期(h)

公式 2 · 多次摄入叠加

A_total(N) = Σ Xᵢ × (1/2)^((N-Tᵢ) / t₁/₂)

Xi = 第i次剂量, Ti = 第i次时间（仅当 N > Ti 时计入）

公式 3 · 一室模型 (含吸收相)

C(N) = (F·X·kₐ)/(Vd·(kₐ-kₑ)) × (e^(-kₑ·N) - e^(-kₐ·N))

半衰期计算 · 个体化模型

function calcHalfLife(profile: UserProfile): number {
    if (profile.customHalfLife != null) {
        return round(profile.customHalfLife, 1);
    }

    // 默认基础半衰期来自用户自评；若提供基因型则优先覆盖
    let base = { fast: 3, normal: 5, slow: 7, unknown: 5 }[profile.metabolismPreset];
    if (profile.cyp1a2Genotype && profile.cyp1a2Genotype !== 'unknown') {
        base = { 'AA': 3, 'AC': 7, 'CC': 10 }[profile.cyp1a2Genotype];
    }
  
  // 影响因素调整
  if (profile.isSmoker)             base *= 0.6;   // 吸烟加速
  if (profile.usesOralContraceptives) base *= 2.0;   // 避孕药减慢
  if (profile.isPregnant)           base *= 3.0;   // 怀孕极慢
  if (profile.drinksAlcohol)        base *= 1.4;   // 饮酒减慢
  if (profile.age > 65)             base *= 1.3;   // 高龄减慢
  
  // 肝功能调整
  const liverMul = { none: 1, mild: 2, moderate: 3, severe: 4 };
  base *= liverMul[profile.liverDisease];
  
    return Math.max(1, round(base, 1));
}

产品输出应统一表述为“估算值”或“风险提示”，而非确定性的安全结论。

7 技术架构

前端

  • 框架：Flutter + Dart
  • 状态管理：MVP 先用 ChangeNotifier / ValueNotifier，后续可升级 Riverpod
  • 图表：CustomPainter / fl_chart（趋势曲线）
  • 组件风格：Material 3 基础上做品牌化定制
  • 目标平台：Android / iOS / Web，多端共用同一套计算逻辑

存储（MVP 本地优先）

  • 设置存储：SharedPreferences
  • 记录存储：Hive / SQLite（二选一）
  • 离线可用：默认支持，本地计算不依赖网络

可选后端 (v2)

  • 认证：Supabase Auth / Firebase Auth
  • 数据库：Supabase PostgreSQL / Firestore
  • 同步：跨设备备份与恢复
